Use Hexo to create static blog

It is getting more and more popular to writing a static blog. Instead of using wordpress, we can use the static site generator like Hexo to generate the html from plain text , and then publish it with Github webpage for free.

This article will introduce how I set up this Hexo blog step by step.

Set up Hexo environment

Hexo is based on node.js, and I used Github to publish the webpage, so make sure you have node.js and git installed.

After node.js is installed , we can use node.js package manager—npm to install Hexo

Create an Hexo project

Once Hexo is installed, we can use Hexo command to initialize some folders/files for the website. Say we will work under <blogFolder>, run the commnand:

Once initialised, the <blogFolder> will looks like:

In this blog, we can only care about:

_config.yaml

This is for website configuration, mostly it will decide how generated static page will be deployed to the webserver.

package.json

Define the Hexo version and the dependencies. Markdown is one of the dependency, so we can use Markfown to write blog.

source

This is where you put your site’s content. Hexo ignores hidden files and files or folders whose names are prefixed with _ (underscore) - except the _posts folder. Renderable files (e.g. Markdown, HTML) will be processed and put into the public folder, while other files will simply be copied.

Other folders are for more customized defined.
A “hello-world” blog comes along with this initial website. You can check it under <blogFolder>/source/_posts. hello-world.md is the Markdown file for the blog.

Generate static webpages and publish to server

Use Hexo command to translate Markdown file to static webpages.

The static page are generated under publish folder. We can start local Hexo server for this pages.

Now you can visit the website(default port is 4000): http://localhost:4000/

The static pages can be deployed on public webserver. Many people use Github webpage as it is free and public .
